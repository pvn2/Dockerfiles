Docker Commands
--------------------
Working on Images
-------------------------
1.To start the docker
   systemctl start docker

2.To check the status of docker
   systemctl status docker

3.To stop the docker 
   systemctl status docker
   To stop the docker completly
   systemctl stop docker.socket
  
4.To download (Pull) an image or a repository from a registry
   docker pull  <image_name>
   (or)
   docker pull <image_name>:<latest>
   Note:  if we dont mentined anything it will download latest version
   docker pull <image_name>:<version_number>
   Note:  it will download the specific version

2 To see the list of docker images 
  docker image ls 
  (or) 
  docker images 

3 To delete a docker image from docker host 
  docker rmi <image_name>/<image_id> 

4) To upload a docker image into docker hub 
   docker push image_name 

5) To tag an image 
   docker tag   image_name   ipaddress_of_local_registry:5000/image_name 

6) To build an image from a customised container 
  docker   commit  container_name/container_id     new_image_name 

7) To create an image from docker file 
   docker build -t    new_image_name 

8) To search for a docker image 
   docker search image_name 

9)  To delete all images that are not attached to containers 
   docker system prune -a 

Working on containers
-----------------------------
 1.To create a container from a docker image
     docker run image_name 
     Note: from image container will be created with default name,if you want to give custom name need to give  below command 
     
     docker run --name <custom-container-name> -p <new-port>:<old-port>  <image-name> 
     docker run --name <custom-container-name> -p <new-port>:<old-port>  -d <image-name>
     Note:
     The above command runs tomcat in detached mode, so we get out # prompt back
     This is check in local system if not find then it will create image and container based on image from dockerhub and it will be            runnig  mode
  	
     Run command option:
     ----------------------------
     -p           Used for port mapping between port of container with the dockerhost port.
     --name   Used for giving a name to a container 
     -d           Used for running the container in detached mode as a background process 
                   (ctr+c) to stop the backgroud process
    
    2.To see the list of running and stopped containers 
    docker   ps -a 

   3.To see the list of all running continers 
    docker  container  ls 

    4.To start a container 
    docker  start  <container_name/container_id> 

    5.To stop a container
    docker stop <container_name/container_id>

    6.To restart a container
    docker restart container_name/container_id 
    
    7.To restart after 10 seconds 
    docker restart  -t  10  container_name/container_id

    8.To delete a stopped container and containers
    docker  rm  container_name/container_id
    docker  rm  container_name_1/container_id container_name-2/container_id  container_name-3/container_id

    9.To delete a running container 
    docker  rm  -f  container_name/container id 

    10.To stop all running containers 
    docker stop $(docker ps -aq) 
 
    11.To restart all containers 
    docker restart $(docker ps -aq) 

    12.To remove all stopped containers 
    docker rm $(docker ps -aq) 

   13.To remove all contianers(running and stopped) 
    docker rm -f  $(docker ps -aq) 
    Note: -f --> stands for file systems
  
   14.To see the logs generated by a container 
   docker logs <container_name/container_id> 

   15.To see the ports used by a container 
   docker port <container_name/container_id> 

   16.To get detailed info about a container 
   docker inspect <container_name/container_id> 

   17.To go into the shell of a running contianer which is moved into background 
   docker attach <container_name/container id>

    18.Direct run command without pull
   docker run --name <container-name> -d -P <image-name>
   docker run --name appserver -d -P nginx
   
   Note:
   -------
   if image not available then it will perform pull operation automatically
   
   Run command option:
   ----------------------------
   -P(capital)      will perform automatic port mapping 
    
     19.To Check port number
     docker port <container-name>
     docker port appserver
   
     80/tcp -> 0.0.0.0:49155
     80/tcp -> :::49155
    
     80 is default port
     49155 is custom port
     
     we can check using (docker container ls)

20.To execute anycommand in a container 
   docker exec -it container_name/container_id command
   Eg: To launch the bash shell in a contianer 
   docker exec -it container_name/container_id    bash 

++++++++++++++++++++++++++++++++++++++++++++++++
Run command options 
-v 		Used for attaching a volume to the container 
--volume-from 	Used for sharing volume between containers 
--network 		Used to run the contianer on a specific network 
--link 		Used for linking the container for creating a multi container architecture 
--memory  	Used to specify the maximum amount of ram that the container can use
